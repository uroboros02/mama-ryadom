# Финмодель новой точки «Мама рядом» — состояние на 06.07.2026

Вся финансовая история проекта собрана в этой папке (`finmodel/`); 06.07 сюда переехали
данные из корня репо, а рабочая папка temp/ расформирована. Инструменты и выводы
восстановимы по памяти Claude (`project-finmodel-new-site.md`), но файлы в `данные/` —
единственные копии выгрузок, их беречь.

## Структура папки

- `finmodel-trenazher.html` — исходник веб-тренажёра (редеплой тем же URL, favicon 🎛️)
- `sim_v3.py` · `ref_v3.py` · `verify_v3.js` — python-эталон движка и проверочная пара
  (после правок HTML: `node verify_v3.js` против `python3 ref_v3.py` — числа обязаны совпасть)
- `dom-test.js` — функциональный тест UI без браузера (заглушка DOM + реальный скрипт страницы;
  проверяет связки микс→чек, отток↔жизнь, предпродажу; запуск: `node dom-test.js`)
- `tilda-finmodel-block.html` (+ `tilda-код-для-вставки.txt` — то же текстом) — изолированный блок для Тильды (T123)
- `Финмодель Мама рядом — тренажёр (локальная копия).html` — свежий снапшот, чтобы слать файлом
- `build_model_v5.py` — генератор Excel (под старый движок v2, НЕ запущен; переписать под v3 по «добру»)
- `данные/` — исходник собственницы + бенчмарки старой точки «Мама и малыш» (МИМ)
- `архив/` — движки и Excel эпохи v2 (история, не трогаем)

   из ЛТВ; принято юзером «для простоты 2»).

   (7 слотов × 2 зала × 6 детей, Пн–Сб) при факте 2 и планке 70% держит ~760 абонементов;
   резерв +12 посещ/день добавляется ТОЛЬКО утром в 9:00 (до 96/день → ~870).

4. Абонементы не замораживаются и не возвращаются, неотхоженное сгорает; деньги
   в модели размазаны по месяцам хождения (реальная касса на росте чуть лучше — предоплата).
5. Жизнь факт = 3 мес (директива юзера; ЛТВ: медиана «месяцев ходили» 2, среднее 2,6).
6. **Воронка по этапам добыта из выгрузки заявок** («заявки авг 24 - июль 26 олимпико.xlsx»,
   SportCRM, 7 970 строк; колонка «Дата/время» = запись на пробное): платные VK —
   заявка→запись 38,5%, запись→зачислен 28%, сквозная 11,0% (n=1105); органика 68/44/30%;
   «без источника» (875 шт) ведёт себя как органика (33%); WA-чаты (5 657) — не лиды,
   операционка действующих. Доходимость запись→пришёл в снимке НЕ видна (статусы
   перезаписываются). Orders-2.xlsx = оплаты (дата+сумма, без ФИО) — для воронки не годится.
   Юзер принял в движок: **лид→пробное 30% × пробное→покупка 40%** (сквозная 12%).
