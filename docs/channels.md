## Каналы (транспорт)

| Канал | Доля трафика | Интеграция | Когда |
|---|---|---|---|
| **MAX** | ~60% | Wazzup (проверить, что отдаёт MAX-канал; иначе MAX API напрямую) | **Старт** |
| WhatsApp | ~30% | Wazzup (QR-привязка — окна 24ч нет) | Фаза 1/2 |
| Telegram | ~5% | Wazzup / Bot API | Позже |
| VK | ~5% | Wazzup / VK API | Позже |
| Телефон | — | Мегафон-АТС (только запись → анализ) | Фаза 2 |

- **Старт с MAX** — самый большой объём = максимум реальных диалогов для итераций.
- Логика агента **не зависит от канала** (роутер). Fitbase Lead хранит идентификаторы
  каналов в полях (`phone, max, vk, telegram, instagram, email`) → один человек = один лид,
  мультиканальная дедупликация на уровне модели данных (+ поле `is_duplicate`).
- **Модель — все каналы сразу** (опознание не зависит от канала, якорь — телефон; чат без
  телефона авто-сшить можем не суметь — край на потом); **транспорт — по мере включения**
  канала (MAX старт, дальше по «Когда» в таблице).
- **Бот не звонит сам.** Звонит продавец, звонок записывается, запись анализируется.
- **VK — два разных входа:** чат сообщества ВК = канал двери A (строка «VK» выше); **VK Ads
  (таргет) лид-формы** = источник двери B (близнец Тильды), в личку не пишем. См. «Две двери
  входящего».
