# Разбор выгрузки переписок Wazzup (`chats wazzup.csv`)

> Анализ сырой выгрузки из Wazzup, сделан 2026-06-06. Источник — общий WhatsApp-номер
> школы (шаблоны подписаны **«Олимпико, центр гимнастики во Дворце молодёжи»**, старая CRM
> `ufagymnastics`). «Мама рядом» (малыши 1.5–4) — программа внутри этой школы. Весь анализ
> локальный, ПД в LLM не уходили. Кластеризация — кейвордами (не семантика), поэтому частоты
> читать как **ранжир тем**, а не точные числа; засорённые корзины помечены ⚠️.

## 1. Что это за данные (объём и форма)

| Параметр | Значение |
|---|---|
| Сообщений (честный CSV-парс) | **100 000** (ровно — почти наверняка **лимит выгрузки**, не вся история) |
| Диалогов (`chat_id` = телефон) | **2 175** |
| Разных людей (`user_name`) | ~1 650 |
| Каналы | **WhatsApp 96.7%**, Telegram 0.1%, MAX/VK — нет |
| Направление | 55% от клиента / 45% наши |
| Глубина диалогов | медиана **19** сообщений; 66% диалогов ≥10 |
| Текст vs медиа | 93% с текстом, 7% картинки/файлы |
| Порядок строк | **от новых к старым** (диалог собирается чтением снизу вверх) |

Файл — это по сути **один WhatsApp-номер** (79272361991; + 132 сообщения Telegram на нём же).
Других линий нет. Поэтому 2 175 диалогов — это срез **одного канала**: лиды, пришедшие через
MAX/иные номера, в файл не попадают (вероятная причина, почему диалогов кажется мало).

### Чего в данных НЕТ (ограничения — важно)
1. **Нет даты/времени ни у одного сообщения** → нельзя мерить скорость ответа (≤5 мин),
   тихие часы, ритм «рваного цикла». Содержание — да, тайминг — нет.
2. **Нет исхода сделки**: связь с CRM (`crm_user_id`) заполнена в **13 строках из 100 000**.
   Видно *как* общались, не видно *кто купил* → «какой заход конвертит лучше» по файлу
   не измерить, нужна стыковка с Fitbase.
3. **Ровно 100k = срез**, не полная база.

### Покрытие по времени (проверено 2026-06-06)
Запрашивали выгрузку **с августа 2024**. Но: (1) ровно 100 000 строк = почти наверняка
**лимит экспорта**; (2) порядок новые→старые, а самые старые строки файла — **конец ноября
2024** (напоминания о справках от 27.11/8.12/15.12, файл «Документ-2024-11-25»); плотность
явных дат в тексте резко нарастает с декабря 2024. → Файл покрывает **~конец ноября 2024 →
июнь 2026**; **август–ноябрь 2024 не попали** (обрезаны лимитом). Потеряли самый ранний
(малоактивный) кусок диапазона; по числу сообщений потеря, скорее всего, небольшая, но
**точную долю по файлу не вычислить** — нужен общий счётчик за период (от разрабов / из Wazzup).

## 2. Главный вывод: это лог OPERATIONS школы, а не лид-инбокс

Самое важное. Заголовочные «100k сообщений» обманчивы: **бо́льшая часть — это переписка с
действующими клиентами и операционка**, а не продажи лидам (= работа бота). Раскладка по 2 175 чатам:

| Срез чатов | Кол-во | Доля |
|---|---|---|
| Есть сигнал лид-воронки (заявка/цена/возраст/пробное) | 614 | 28% |
| Есть сигнал operations (кабинет/чеки/разряд/тренер) | 340 | 16% |
| └ только лид (чистая зона бота) | 429 | 20% |
| └ только operations (точно не бот) | 155 | 7% |
| └ и то и то | 185 | 9% |
| Ни одного сигнала (реакции/болтовня/логистика) | 1 406 | 65% |

На уровне сообщений: лид-сигналов ~**1 426** из 50 902 входящих (**2.8%**), ops-сигналов ~2 058.

**Качественные маркеры ops (однозначные):** «почему в кабинете не отображается расписание»,
«возврат налога за 2024–2025», «чеки запрошу с банка», «в связи с увольнением Басыровой А.Р.,
какого тренера назначат?», «первый юношеский разряд», «первенство», «девочки 2018 г.р.».
→ Школа ведёт и **серьёзную спортивную гимнастику для старших**; «Мама рядом» (1.5–4) — узкий сегмент.

**Следствие для проекта:** для FAQ/скриптов нужно сперва **отфильтровать лид-слой**; сырой
агрегат доминирован операционкой и будет вводить в заблуждение. Объём именно лид-продаж —
порядка **нескольких сотен диалогов**, не 2 175.

## 4. Скрипты продаж — реальный playbook менеджеров (исходящие по стадиям)

| Стадия | сообщ. | Доля | Реальные формулировки |
|---|---|---|---|
| Приглашение на пробное | 11 067 | 26.5% | «есть место к Ирине Анатольевне. Записать вас?» |
| Закрытие / оплата / ЛК | 7 644 | 18.3% | «Инструкция по оплате…», «создала вам личный кабинет» |
| Дожим / фоллоу-ап | 7 083 | 16.9% | «очень ждём обратной связи», «места занимают активно» |
| Первичный контакт | 5 205 | 12.4% | «Это Айна из центра гимнастики Олимпико…» |
| Цена / оффер / акция | 3 346 | 8.0% | «на июнь доступен абонемент на 4 занятия» |
| Пост-пробное (догрев) | 671 | 1.6% | «были на пробном, как эмоции, всё понравилось?» |
| **Квалификация (имя/возраст ребёнка)** | **22** | **0.1%** | «сколько лет ребёнку?» |

**Инсайт №1:** менеджеры **почти не квалифицируют** (возраст спрашивают в 22 сообщениях из
42k) — сразу прыгают на запись пробного. Структурированная квалификация бота (F1+F2
react-qualify) — это **реальное усиление**, а не просто паритет с человеком.

**Инсайт №2:** топ-активность = приглашение на пробное + дожим + закрытие оплаты — ровно
до-КЭВ и после-КЭВ работа бота. Скрипты уже есть, их можно поднять в промпт почти как есть.

## 5. Крайние случаи (зона бота)

| Тип | ~сообщ. | Реальные примеры |
|---|---|---|
| Отъезд / отпуск / переезд | 287 | «уехали в деревню, как приедем — запишусь», «не успели на ЭКГ, уехали, подождёте до августа?» |
| Уход / передумали | 169 | «передумали», «малышка совсем, пока не будем, и ездить далеко» |
| Дорого / финансы | 114 | «пока нет возможности оплатить, возможно вернусь в начале июня» |
| Сомнение / решение в семье | 111 | «всё понравилось, по абонементу пока думаем», «обсудим с мужем» |
| ⚠️ «Возврат» | 141 | в основном **возврат налога**, не жалобы-возвраты (корзину чистить) |

**Инсайт №3:** лид-фрикшн — это «вернусь позже», «обсудим с мужем», «уехали — напишу как
приедем». Это ровно те кейсы долгого рваного цикла, ради которых нужны **проактивная память
и касания** бота. Живое подтверждение, что проактив-дизайн не теоретический.

## 6. Что со всем этим делать дальше

1. **FAQ для бота** можно собирать уже сейчас — из лупы B (лид-слой) + лупа A для операционных
   тем, которые тоже прилетают. Но писать FAQ бот будет сам из `docs/config.md` — этот разбор
   даёт *перечень тем и тон вопросов*, а не готовые ответы.
2. **Скрипты** (раздел 4) — материал для few-shot/тона в промпте F1.
3. **Фильтр лид-слоя — СОБРАН** → `data/leads-filtered.md` (2026-06-06): **248 чистых лидов**
   (намерение клиента, без операционки) + **265 «лид→клиент»** (виден лид-участок). Лид ловим
   по ВХОДЯЩИМ (слова клиента), действующих клиентов отсекаем по сервисным маркерам в любую
   сторону. ПД замаскированы, реплики в хронологии. Высокая точность / низкая полнота —
   немногословные лиды, не набравшие ключевых слов, остаются в «ни то ни другое» (1 081).
4. **Для метрик конверсии** — отдельная стыковка с Fitbase (этот файл исхода не содержит).
5. **Перевыгрузка** — если нужен конкретный период или вся история (текущий файл обрезан на 100k).
