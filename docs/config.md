# Конфиг бота (фактбук) — данные, правятся без кода

> Всё, что бот **читает из контекста** (Шаг 4, секция C) и озвучивает живым языком: продукты,
> возраст, цены, скидки, расписание, адрес, тексты. **Это данные, а не код** — меняются без релиза.
> Бот их **озвучивает, но не выдумывает**: нет факта здесь → не сочиняет (Шаг 6: сигнал/эскалация).
> FAQ бот **пишет сам** своими словами (по правилам продаж из промпта), опираясь на эти факты —
> заготовленных ответов не держим. Данные **утверждены заказчиком 2026-06-02** (собраны 2026-06-01, заказчик подтвердил без
> изменений). Расписание на волне 1 — **намеренно тестовое (придуманное)**, реальное позже из
> Fitbase; контакты и видео — пока плейсхолдеры (заменить перед боевым запуском). В коде станет YAML/таблицей.

## Направления (3)
| Направление | Пол | Возраст |
|---|---|---|
| **Гимнастика** | любой | 1,5–4 включительно |
| **Беби балет** | только девочки (мальчику → не отказ, альтернатива) | 1,5–4 |

Описания (чтобы бот описал точно, не выдумал):
- **Гимнастика** — полосы препятствий, мягкие модули, баланс, лазание, прыжки, мячи, крупная моторика и координация.
- **Беби балет** — ритмика под классическую музыку: чувство ритма, мягкая растяжка, пластика, осанка, лёгкая хореография.
- **Сенсорика** — вода, песок, крупы, лёд, текстуры: мелкая моторика, пальчиковые игры, простые слова, звукоподражания, песни.

## Уровни (по возрасту)
- **1,5–3** → с мамой обязательно
- **3–4** → с мамой опционально (4 включительно)

## Края (квалификация по возрасту)
- **Младше 1,5** — НЕ отказ. Берём, если со слов мамы: уверенно ходит сам · спокоен с новыми людьми/детьми, готов к группе · мама готова. Предупредить: нет пеленальных столов и места для кормления. Не подходит → **Резерв** + напомнить написать к ~1,4 года.
- **5+** — мягкая рамка, НЕ отказ: «прийти можно, но программа примерно до 4 — в 5 может быть просто неинтересно».

## Вместимость
- Размер группы: **до 6 пар** — параметр, меняется без кода.
- Открываемся с нуля → места есть везде; «держать 1 место под пробную» — будущая настройка, **пока выкл**.
- Нет места в нужном окне → **не Резерв**, а предложить **другое окно** со свободным местом.
- Резерв = только возрастной край (<1,5), не за вместимость.

## Этапы воронки (F1)
- **Неразобранные** — заявка, которой ещё не писали / не реагировали.
- **В работе** — как только бот написал первым ИЛИ ответил (любая коммуникация); двигает код. «Подходит» → остаётся здесь, ведём к записи.
- **Резерв** — возрастной край (<1,5 не готов).
- **Архив** — не наш профиль (бот предлагает, человек подтверждает).

- **Индивидуальные:**
  - разовое — **2500** · 4 зан./1 мес — **9000** · 8 зан./2 мес — **16000**

## Скидки (бот применяет только из этого списка; вне списка → эскалация)
- **−15%** при двоих детях — на **абонемент**.
- **10% «привёл друга»** — обоим (новому + приведшему). Только на **абонемент или разовую**, **не на пробное**. Условие: новый называет **номер** приведшего → **код** ищет в базе действующих клиентов; не нашёл → алерт + задача продавцу. *(Механика — задача в [`tasks-open.md`](tasks-open.md) §F3.)*
- 🔴 **Бот НЕ предлагает скидки сам** — не рекламирует, не «у нас ещё есть такая, давайте её» (защита маржи). Применяет **только ту, которую клиент сам упомянул** / на которую сам претендует.
- **Не суммируются** — одна скидка за раз (та, что клиент назвал).
